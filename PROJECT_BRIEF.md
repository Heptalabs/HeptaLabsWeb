# PROJECT_BRIEF

## 프로젝트 요약 (KR)
HeptaLabs Web는 HeptaLabs의 회사 소개를 위한 정적 웹사이트 프로젝트다. 본 단계의 목적은 프론트 코드 대규모 개편이 아니라, 향후 작업 속도를 높이는 문서 체계를 확립하는 것이다.

## 목표
- 브랜드 소개 메시지를 일관되게 전달
- 문의 전환을 위한 핵심 CTA를 명확히 배치
- 사람용/에이전트용 문서를 분리해 운영 효율 향상

## 타깃 사용자
- 잠재 고객(기업/파트너)
- 외부 협업자(디자이너, 개발자, 마케터)
- AI 코딩 에이전트(문서 기반 작업 자동화)

## 핵심 메시지
- 복잡한 비즈니스 과제를 구조화해 실행 가능한 형태로 전환
- 전략과 실행을 분리하지 않고 하나의 흐름으로 설계
- 빠른 실행과 신뢰 가능한 커뮤니케이션을 동시 확보

## 성공 지표 (문서/운영 관점)
- 신규 기여자가 README만으로 로컬 실행에 성공
- 에이전트 작업 시 AGENTS.md 기준으로 일관된 결과 도출
- 문서 간 링크 불일치/중복 규칙 최소화

## 출처 기반 요약 메모
AGENTS.md 관련 영상/글의 핵심은 다음과 같다.
- README는 사람 중심
- AGENTS.md는 에이전트 실행 중심
- 단일 지침 파일로 규칙 분산 문제를 줄임
- 기존 도구 호환이 필요하면 AGENT.md를 포인터로 유지

## 中文摘要
- HeptaLabs Web 是公司介绍型静态网站项目，本阶段重点是“文档体系完善”而非大规模前端重构。
- 目标是统一品牌信息、提高咨询转化效率，并通过 README/AGENTS 分层降低协作成本。
- 成功标准包括：新成员可快速本地运行、代理执行一致、文档规则无冲突。

## English Summary
- HeptaLabs Web is a static company website project; this phase focuses on documentation architecture, not major frontend redesign.
- Goals: consistent brand message, stronger inquiry CTA clarity, and lower collaboration cost through README vs AGENTS separation.
- Success criteria: fast onboarding, consistent agent behavior, and minimal doc-rule conflicts.
