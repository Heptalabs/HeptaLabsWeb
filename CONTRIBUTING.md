# CONTRIBUTING

## 협업 원칙 (KR)
이 저장소는 작은 변경을 빠르게 누적하는 방식으로 운영한다. 모든 기여는 문서 또는 UI 메시지 품질을 높이는 것을 목표로 한다.

## 브랜치 네이밍
- `docs/<topic>`: 문서 추가/수정
- `content/<topic>`: 카피/메시지 변경
- `fix/<topic>`: 오류 수정

예시:
- `docs/add-deployment-checklist`
- `content/refine-hero-cta`
- `fix/readme-broken-link`

## 커밋 컨벤션
- `docs: ...`
- `content: ...`
- `fix: ...`

예시:
- `docs: add AGENTS and deployment docs`
- `content: refine services copy matrix`

## PR 체크리스트
- 변경 목적이 명확한가?
- 사용자/팀에 미치는 영향이 설명됐는가?
- 링크/앵커가 정상 동작하는가?
- 시각 변경 시 스크린샷이 포함됐는가?
- README, AGENTS, DEPLOYMENT 규칙과 충돌하지 않는가?

## 리뷰 기준
- 정확성: 사실/명령/절차 오류가 없는가?
- 일관성: 용어, 톤, 구조가 기존 문서와 맞는가?
- 실행성: 새로운 팀원이 바로 따라 할 수 있는가?

## 금지 사항
- 근거 없는 성과 수치 추가
- 파괴적 Git 명령 사용(명시적 요청 없는 강제 push 포함)
- 타 문서 규칙과 상충되는 독립 규칙 추가

## 中文摘要
- 采用小步快跑的协作方式，所有贡献应提升文档或站点信息质量。
- 使用统一分支/提交前缀，PR 必须包含目的、影响与验证结果。
- 审查重点为准确性、一致性与可执行性。

## English Summary
- We follow small, incremental collaboration to improve doc and messaging quality.
- Use consistent branch and commit conventions; PRs must include purpose, impact, and validation notes.
- Review focuses on accuracy, consistency, and execution clarity.
